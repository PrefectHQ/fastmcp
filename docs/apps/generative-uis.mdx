Prefab components compile to a JSON protocol — your Python builds a component tree, it serializes to JSON, and the renderer interprets it. The renderer doesn't care where the JSON came from. An LLM can produce it directly.

This means you can create a tool that accepts component JSON as its argument and renders whatever the model describes. The model decides at runtime whether to show a bar chart, a form, a data table, or a combination — adapting the UI to the conversation. And because the JSON protocol includes actions like `ToolCall` and `SetState`, generated UIs can be fully interactive.

## How It Works

Create a tool whose argument is the component JSON, and return it as structured content:

```python
from prefab_ui.app import PrefabApp
from fastmcp import FastMCP

mcp = FastMCP("Generative UI")


@mcp.tool(app=True)
def render_ui(ui: dict, state: dict | None = None) -> PrefabApp:
    """Render a Prefab UI from a JSON component tree.

    The `ui` argument should be a valid Prefab component JSON object.
    The optional `state` argument provides initial client-side state values.
    """
    app = PrefabApp.from_json({"view": ui, "state": state or {}})
    return app
```

The model calls this tool with a component tree as the `ui` argument. The user sees whatever the model described.

<Info>
  `PrefabApp.from_json()` and built-in generative UI tooling are under active development. The concept works today with manual JSON construction, and we're building ergonomic helpers to make it seamless.
</Info>

## Use Cases

**Adaptive presentation.** The model examines data and picks the best visualization — line chart for trends, pie chart for proportions, table for detailed records. The presentation adapts to the content.

**Dynamic forms.** The model generates a form tailored to the conversation — a bug report form with severity and repro steps, or a deployment form with region and instance type. The form structure becomes part of the model's reasoning.

**Guided workflows.** The model generates a UI, the user interacts with it, and the model generates a new UI based on that interaction. Each step builds on the last.

## The JSON Protocol

Every component has a `type`, optional `props`, and optional `children`:

```json
{
  "type": "Column",
  "props": { "gap": 4, "cssClass": "p-6" },
  "children": [
    {
      "type": "Heading",
      "props": { "text": "Hello, World" }
    },
    {
      "type": "Text",
      "props": { "text": "This UI was generated by the model." }
    }
  ]
}
```

Every Python component has a corresponding JSON representation. See the [Prefab UI documentation](https://prefab.prefect.io) for the full protocol reference.
