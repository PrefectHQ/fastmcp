MCP tools return text. That works for simple answers, but a surprising amount of tool output — query results, metrics, lists of records, configuration options — is information that humans consume better visually than as a wall of text in a chat message.

Without Apps, that data has only one place to go: the LLM's context window. A database query returns 500 rows? They all go into context. An API returns a nested JSON response? The model has to parse it and summarize. The human never sees the raw data, and the model burns tokens presenting it.

MCP Apps change this. A tool can return an interactive UI — a chart, a sortable table, a form — rendered in a sandboxed iframe right inside the host client's conversation. The data goes directly to the human in a format designed for human consumption, while the model can focus on what it's good at.

FastMCP implements the [MCP Apps extension](https://modelcontextprotocol.io/docs/extensions/apps) and provides two approaches:

## Prefab UI (Recommended)

<VersionBadge version="3.1.0" />

[Prefab UI](/apps/prefab) is a Python component library that compiles to a JSON protocol, rendered by a shared engine that FastMCP manages automatically. You describe your UI in Python — layouts, charts, tables, forms — and FastMCP handles the renderer, the protocol wiring, and the delivery.

```python
from prefab_ui.components import Column, Heading, BarChart, ChartSeries
from prefab_ui.app import PrefabApp
from fastmcp import FastMCP

mcp = FastMCP("Dashboard")

@mcp.tool(app=True)
def sales_chart(year: int) -> PrefabApp:
    """Show sales data as an interactive chart."""
    data = get_sales_data(year)

    with Column(gap=4, css_class="p-6") as view:
        Heading(f"{year} Sales")
        BarChart(
            data=data,
            series=[ChartSeries(data_key="revenue", label="Revenue")],
            x_axis="month",
        )

    return PrefabApp(view=view)
```

No HTML, no JavaScript, no frontend build step. Install with `pip install "fastmcp[apps]"` and see [Prefab UI](/apps/prefab) for the full guide.

## Low-Level API

The [low-level API](/apps/low-level) gives you full control when you need it. You write the HTML yourself, wire up communication with the host via the `@modelcontextprotocol/ext-apps` JavaScript SDK, and manage resources and CSP configuration directly.

This is the right choice when you need custom rendering (maps, 3D, video), want to use a specific JavaScript framework, or need capabilities beyond what the component library offers.

```python
from fastmcp import FastMCP
from fastmcp.server.apps import AppConfig, ResourceCSP

mcp = FastMCP("Custom App")

@mcp.tool(app=AppConfig(resource_uri="ui://my-app/view.html"))
def my_tool() -> str:
    return '{"values": [1, 2, 3]}'

@mcp.resource(
    "ui://my-app/view.html",
    app=AppConfig(csp=ResourceCSP(resource_domains=["https://unpkg.com"])),
)
def view() -> str:
    return "<html>...</html>"
```

See [Low-Level API](/apps/low-level) for the full reference.
